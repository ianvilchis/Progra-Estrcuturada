## Descripción General
Este repositorio constituye el portafolio digital de evidencias para la materia de Programación Estructurada. Contiene el conjunto de prácticas de laboratorio, algoritmos modulares y proyectos desarrollados a lo largo del ciclo escolar en el lenguaje C. El objetivo de este espacio es demostrar de forma práctica el dominio de la metodología estructurada, transitando desde la manipulación básica de operadores y flujos de control, hasta la gestión avanzada de la memoria RAM y la persistencia de datos en almacenamiento secundario.

## Estructura del Repositorio
El proyecto se encuentra organizado en carpetas consecutivas alineadas estrictamente con las actividades y bloques temáticos evaluados en la plataforma Classroom, incluyendo subcarpetas estandarizadas de `codigo/` y `documentacion/` según los lineamientos requeridos:

* **📂 actividad_01_operadores/**: Prácticas iniciales enfocadas en tipos de datos primitivos, asignación de variables y operaciones matemáticas secuenciales.
* **📂 actividad_02_condicionales/**: Estructuras de selección lógica (`if`, `else`, `switch`) aplicadas a la toma de decisiones del programa.
* **📂 actividad_03_ciclos/**: Implementación de estructuras de control repetitivas (`for`, `while`, `do-while`) para la optimización de tareas iterativas.
* **📂 actividad_04_arreglos/**: Manipulación de estructuras de datos estáticas homogéneas, destacando el desarrollo de matrices bidimensionales complejas a través de un juego interactivo de Batalla Naval.
* **📂 actividad_05_funciones/**: Aplicación de modularidad mediante la definición de prototipos, funciones independientes y paso de parámetros, integrando criterios de filtrado de datos.
* **📂 actividad_06_punteros/**: Estudio de bajo nivel de la memoria RAM, análisis hexadecimal de direcciones físicas y uso de funciones modificadoras junto con asignación dinámica (`malloc`).
* **📂 actividad_07_estructuras/**: Modelado de tipos de datos heterogéneos personalizados (`struct`) para la gestión unificada de registros complejos en un sistema escolar.
* **📂 actividad_08_archivos/**: Implementación de persistencia de datos mediante flujos de escritura (`"w"`), lectura (`"r"`) y adición incremental (`"a"`) de archivos físicos de texto plano (`.txt`).
